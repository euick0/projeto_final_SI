Perguntas de Redes e Apache

TÓPICO 01

1. O que é um endereço IPv4 e como se identifica no Windows?

O IPv4 é um rótulo numérico de 32 bits que identifica cada dispositivo na rede. No Windows, utiliza o comando ipconfig no CMD (Prompt de Comando) para visualizar os detalhes da sua interface de rede.

TÓPICO 02

2. Porque outro PC consegue aceder ao Apache através do IP?

Isso ocorre porque o IP aponta diretamente para a máquina onde o serviço Apache está a "ouvir" numa porta específica (geralmente 80 para HTTP), processando a requisição e devolvendo os ficheiros do site via TCP/IP.

TÓPICO 03

3. Diferença entre localhost e 192.168.x.x?

localhost (127.0.0.1) é o loopback interno da própria máquina; já o endereço 192.168.x.x é a identidade da interface física na rede local, visível para outros nós.

TÓPICO 04

4. O que pode impedir o acesso LAN?

Geralmente o Windows Firewall bloqueia as portas 80/443 por padrão, ou configurações de isolamento de cliente (AP Isolation) no roteador impedem a comunicação entre dispositivos.

TÓPICO 05

5. Como testar num segundo PC da sala?

Assegure que ambos estão no mesmo Wi-Fi/LAN, descubra o IP do host com ipconfig e escreva-o diretamente no navegador do segundo PC. Se não carregar, revise as regras de firewall.
